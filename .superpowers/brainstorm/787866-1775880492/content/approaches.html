Abordagens de Implementação

Escolha como o Supabase se integrará ao seu ecossistema TDS.

Recomendado

1. The Clean Slate

Supabase como Core. O LMS Lite Dashboard (Vite) fala diretamente com o Supabase via SDK. O n8n apenas "carimba" a entrada do aluno via WhatsApp.

  • ✅ Velocidade: Real-time nativo e sem delay de intermediários.
  • ✅ Simplicidade: Aposenta o lms_lite_api.py (Python).
  • ⚠️ Segurança: Exige configuração de RLS (Row Level Security) no Postgres.

2. Orchestrated Flow

n8n como Gateway. O Dashboard não fala com o Supabase. Ele chama Webhooks do n8n, que então consultam o Supabase.

  • ✅ Controle: Lógica de negócio complexa fica centralizada no n8n.
  • ⚠️ Latência: Mais lento (Dashboard → n8n → Supabase → n8n → Dashboard).
  • ⚠️ Complexidade: Muitos workflows para gerenciar cada ação simples.

3. Hybrid Edge

Edge Functions. Usar Supabase Edge Functions (Deno) para substituir scripts Python e partes do n8n.

  • ✅ Moderno: Tudo dentro da stack do Supabase.
  • ⚠️ Curva de Aprendizado: Exige aprender Deno/TypeScript para as funções de borda.
  • ⚠️ Overkill: Pode ser complexo demais para o estágio atual do TDS.

Por que a Opção 1 (Clean Slate)?

É a forma nativa de usar o Supabase. Você ganha o banco de dados, a autenticação e o real-time de uma só vez. O aluno completa uma aula no WhatsApp → o n8n atualiza o Supabase → o Dashboard do aluno brilha na hora sem que ele precise dar F5.
