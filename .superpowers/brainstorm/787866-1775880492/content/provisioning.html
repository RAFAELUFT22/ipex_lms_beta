Provisionamento do Supabase

Como você prefere rodar o Supabase no seu ecossistema TDS?

A

Supabase Cloud (PaaS)

Recomendado. Rápido, grátis para começar, sem erros de "400 Bad Request" no seu servidor. Conecta via API.

Ideal para o LMS Lite focar em funcionalidades.
B

Self-Hosted (Docker)

Tentaremos corrigir o erro de provisionamento no seu servidor. Mais complexo de manter, mas tudo fica sob seu controle.

Exige configuração fina de memória e redes no Dokploy.

Prós Cloud

  • Zero manutenção
  • Confiabilidade alta
  • Auth e Storage pronto

Contras Self-Hosted

  • Consome RAM (CPU) local
  • Backup sob sua responsabilidade
  • Erros de rede como o que você teve
