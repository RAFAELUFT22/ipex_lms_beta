Design Detalhado: Infra & Schema

Seção 1: O Alicerce do TDS-LMS Lite

Plano de Deploy (Nativo)
  1. git clone do repo oficial do Supabase Docker.
  2. Ajuste do .env interno:
    • KONG_HTTP_PORT=8001
    • SITE_URL=ops.tds.cloud
  3. Injeção da rede kreativ_education_net no docker-compose.
  4. docker compose up -d no terminal.
✅ Resolve Erro 400 do Dokploy
Modelo de Dados (Postgres)
  • profiles: id, whatsapp (unique), name, role, metadata.
  • courses: id, slug, title, category, description, active.
  • enrollments: id, student_id, course_id, status (ativo/concluido), progress_percent.
  • learning_logs: id, student_id, type (quiz_answer, lesson_read, catraca_trigger), payload (JSONB), created_at.

Fluxo da "Audit Table" (DLQ/Reporting)

Toda vez que o n8n recebe um evento do WhatsApp, ele grava primeiro na learning_logs. Se o processamento da catraca falhar, o log já está lá para auditoria e retentativa manual/automática.
